# 수정지시서 002: Generator Target 추상화 — 다중 프론트엔드 프레임워크 슬롯

## 배경

fullend는 현재 React TSX만 생성하지만, 장기적으로 Vue, Svelte, Angular 등 다양한 프레임워크를 지원해야 한다.
STML의 parser가 생성하는 IR (`PageSpec`, `FetchBlock`, `ActionBlock`, `ChildNode`)은 이미 프레임워크 중립적이다.
그러나 generator는 React TSX를 직접 string-build하므로 (useQuery, useMutation, JSX 문법 등), 다른 프레임워크를 추가하려면 generator 전체를 복사해야 한다.

**목표:** generator를 Target 인터페이스 기반으로 재구조화하여, 새 프레임워크를 추가할 때 해당 Target만 구현하면 되는 구조를 만든다.

---

## 1. Target 인터페이스 정의

### 현재

```go
// generator/generator.go
func Generate(pages []parser.PageSpec, specsDir, outDir string, opts ...GenerateOptions) (*GenerateResult, error)
func GeneratePage(page parser.PageSpec, specsDir string, opts ...GenerateOptions) string
```

모든 함수가 React TSX를 직접 생성한다. 프레임워크 선택 여지가 없다.

### 변경

```go
// generator/target.go  ★ NEW
package generator

import "github.com/park-jun-woo/stml/parser"

// Target은 특정 프론트엔드 프레임워크의 코드 생성기가 구현해야 하는 인터페이스이다.
type Target interface {
    // GeneratePage는 하나의 페이지를 타겟 프레임워크 소스 코드로 변환한다.
    GeneratePage(page parser.PageSpec, specsDir string, opts GenerateOptions) string

    // FileExtension은 생성 파일의 확장자를 반환한다. (예: ".tsx", ".vue", ".svelte")
    FileExtension() string

    // Dependencies는 이 Target이 필요로 하는 NPM 의존성을 반환한다.
    // 페이지 분석 결과에 따라 동적으로 결정된다.
    Dependencies(pages []parser.PageSpec) map[string]string
}
```

### 기존 함수 시그니처 유지 (하위 호환)

```go
// generator/generator.go — 기존 시그니처 유지, 내부 위임
func Generate(pages []parser.PageSpec, specsDir, outDir string, opts ...GenerateOptions) (*GenerateResult, error) {
    return GenerateWith(DefaultTarget(), pages, specsDir, outDir, opts...)
}

func GeneratePage(page parser.PageSpec, specsDir string, opts ...GenerateOptions) string {
    opt := DefaultOptions()
    if len(opts) > 0 {
        opt = mergeOptions(opt, opts[0])
    }
    return DefaultTarget().GeneratePage(page, specsDir, opt)
}

// GenerateWith는 지정된 Target으로 코드를 생성한다.
func GenerateWith(t Target, pages []parser.PageSpec, specsDir, outDir string, opts ...GenerateOptions) (*GenerateResult, error) {
    opt := DefaultOptions()
    if len(opts) > 0 {
        opt = mergeOptions(opt, opts[0])
    }

    if err := os.MkdirAll(outDir, 0755); err != nil {
        return nil, err
    }

    for _, page := range pages {
        src := t.GeneratePage(page, specsDir, opt)
        ext := t.FileExtension()
        outName := strings.TrimSuffix(page.FileName, ".html") + ext
        outPath := filepath.Join(outDir, outName)
        if err := os.WriteFile(outPath, []byte(src), 0644); err != nil {
            return nil, err
        }
    }

    return &GenerateResult{
        Pages:        len(pages),
        Dependencies: t.Dependencies(pages),
    }, nil
}

// DefaultTarget은 React Target을 반환한다.
func DefaultTarget() Target {
    return &ReactTarget{}
}
```

## 2. React Target 구현 — 기존 코드 이동

### 현재

generator.go, imports.go, templates.go에 React TSX 생성 로직이 직접 작성되어 있다:
- `GeneratePage()` — JSX 렌더링, React hooks, import 구성
- `imports.go` — `useQuery`, `useMutation`, `useForm`, `useState`, `react-router-dom` import
- `templates.go` — JSX 문법 (`<div>`, `{data.map(...)}`, `<form onSubmit={...}>`)
- `collectImports()` — React 고유 hook 분석 (`importSet.useQuery`, `.useMutation` 등)

### 변경

기존 코드를 `ReactTarget` struct의 메서드로 이동한다.

```
generator/
├── target.go              ★ NEW: Target 인터페이스 정의
├── generator.go           수정: 하위 호환 래퍼 + GenerateWith()
├── react_target.go        ★ RENAME/이동: 기존 GeneratePage 로직
├── react_imports.go       ★ RENAME: imports.go → react_imports.go
├── react_templates.go     ★ RENAME: templates.go → react_templates.go
└── generator_test.go      수정: 기존 테스트 유지 (API 불변)
```

```go
// generator/react_target.go  ★ NEW (기존 generator.go에서 이동)
package generator

type ReactTarget struct{}

func (r *ReactTarget) FileExtension() string { return ".tsx" }

func (r *ReactTarget) GeneratePage(page parser.PageSpec, specsDir string, opts GenerateOptions) string {
    // 기존 GeneratePage() 본문 그대로
}

func (r *ReactTarget) Dependencies(pages []parser.PageSpec) map[string]string {
    deps := map[string]string{}
    for _, page := range pages {
        if hasFetchOrAction(page) {
            deps["@tanstack/react-query"] = "^5"
        }
        if hasFormAction(page) {
            deps["react-hook-form"] = "^7"
        }
        if hasRouteParams(page) {
            deps["react-router-dom"] = "^6"
        }
    }
    return deps
}
```

### 이동 대상

| 현재 위치 | 이동 후 |
|---|---|
| `generator.go:GeneratePage()` | `react_target.go:(*ReactTarget).GeneratePage()` |
| `imports.go:collectImports()` | `react_imports.go` (패키지 내부 함수) |
| `imports.go:renderImports()` | `react_imports.go` |
| `imports.go:importSet` struct | `react_imports.go` |
| `templates.go:renderUseQuery()` | `react_templates.go` |
| `templates.go:renderUseMutation()` | `react_templates.go` |
| `templates.go:renderFormHook()` | `react_templates.go` |
| `templates.go:renderChildNodes()` | `react_templates.go` |
| `templates.go:renderFetchJSX()` 등 | `react_templates.go` |
| `templates.go:renderFilterUI()` 등 Phase 5 | `react_templates.go` |

### 유지할 함수 (프레임워크 중립)

| 함수/타입 | 이유 |
|---|---|
| `Generate()` | 하위 호환 래퍼 |
| `GenerateWith()` | Target 기반 진입점 |
| `DefaultTarget()` | ReactTarget 팩토리 |
| `GenerateOptions` | API import 경로 등 공통 설정 |
| `GenerateResult` | 공통 반환 타입 |
| `toLowerFirst()`, `toUpperFirst()` | 범용 유틸 |
| `toComponentName()` | 범용 유틸 |

## 3. GenerateOptions 확장

Target 추상화에 따라 `GenerateOptions`에 Target 지정 필드를 추가한다:

```go
type GenerateOptions struct {
    APIImportPath string  // 기존: API import 경로
    UseClient     bool    // 기존: 'use client' 디렉티브
    Target        Target  // ★ NEW: nil이면 DefaultTarget() 사용
}
```

이렇게 하면 fullend가 옵션으로 Target을 넘길 수도 있다:

```go
stmlgenerator.Generate(pages, specsDir, outDir, stmlgenerator.GenerateOptions{
    APIImportPath: "../api",
    UseClient:     false,
    Target:        &stmlgenerator.ReactTarget{}, // 명시적
})
```

**대안:** `GenerateWith()`와 `GenerateOptions.Target` 중 하나만 있으면 된다.
둘 다 지원하되 `GenerateWith()`가 우선이고, `Generate()` 호출 시 `opts.Target`을 사용하는 것도 가능하다.
이 판단은 구현 시 결정한다.

## 4. 차후 프레임워크 추가 예시

```go
// generator/vue_target.go (미래)
package generator

type VueTarget struct{}

func (v *VueTarget) FileExtension() string { return ".vue" }

func (v *VueTarget) GeneratePage(page parser.PageSpec, specsDir string, opts GenerateOptions) string {
    // PageSpec IR → Vue SFC 생성
    // <template>, <script setup>, <style>
    // page.Fetches → useFetch() 또는 composable
    // page.Children → template 문법 (v-for, v-if)
}

func (v *VueTarget) Dependencies(pages []parser.PageSpec) map[string]string {
    return map[string]string{
        "vue":        "^3",
        "vue-router": "^4",
        "@tanstack/vue-query": "^5",
    }
}
```

fullend 측 호출:

```go
target := generator.DefaultTarget() // React
// 또는
// target := &generator.VueTarget{} // Vue

generator.GenerateWith(target, pages, specsDir, outDir, opts)
```

## 5. 추상화 경계 — 무엇이 공통이고 무엇이 프레임워크 고유인가

### 공통 (generator 패키지 루트에 유지)

| 항목 | 이유 |
|---|---|
| Target 인터페이스 | 모든 프레임워크가 구현 |
| GenerateOptions | API 경로 등 공통 설정 |
| GenerateResult | 의존성 목록 등 공통 반환 |
| 파일 I/O (mkdir, writeFile) | 프레임워크 불문 |
| 유틸 함수 (케이스 변환) | 프레임워크 불문 |

### React 고유 (react_*.go로 이동)

| 항목 | 이유 |
|---|---|
| JSX 문법 | React 전용 |
| useQuery, useMutation hooks | React/TanStack Query 전용 |
| useForm hook | react-hook-form 전용 |
| useState, useParams | React hooks 전용 |
| 'use client' 디렉티브 | Next.js (React 생태계) 전용 |
| importSet struct | React import 체계 전용 |
| `<Routes>`, `<Route>` | react-router-dom 전용 |

### 판단 보류

| 항목 | 비고 |
|---|---|
| custom.ts import | 프레임워크별 custom handler 방식이 다를 수 있음 |
| Tailwind 클래스 | CSS 프레임워크는 별도 축으로 분리 가능 |

## 하위 호환성

**외부 API 변경 없음:**
- `Generate()` — 시그니처 동일, 내부만 DefaultTarget 위임
- `GeneratePage()` — 시그니처 동일
- `GenerateOptions` — 기존 필드 유지, Target 필드 추가 (zero value = nil = DefaultTarget)
- `GenerateResult` — 동일

**추가 API:**
- `GenerateWith(target Target, ...)` — 새 진입점
- `DefaultTarget() Target` — ReactTarget 팩토리
- `Target` 인터페이스 — 새 타입
- `ReactTarget` struct — 새 타입

기존 fullend 코드는 수정 없이 동작한다.

## 우선순위

**중간** — 현재 React만 지원하므로 즉시 필요하지 않다. 그러나 구조 변경은 코드가 커지기 전에 하는 것이 비용이 적다.
기존 테스트가 모두 통과하면 위험도가 낮은 리팩토링이다.
