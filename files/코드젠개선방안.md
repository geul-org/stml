# 코드젠 개선 방안

> 현재 코드젠은 API 바인딩(what)은 정확하지만, UI 렌더링(how)이 빈약하다. STML의 HTML 원본에 이미 담겨 있는 정보를 코드젠이 버리고 있다.

## 원칙

STML HTML에 작성된 모든 시각적 의도를 TSX에 보존한다. 코드젠이 추가하는 것은 훅과 바인딩 로직뿐이고, 나머지는 원본 그대로 옮긴다.

## 개선 항목

### 1. HTML 구조 보존

**현재**: 모든 요소가 `<div>`로 치환됨
**개선**: 원본 태그(`<header>`, `<section>`, `<footer>`, `<article>`, `<ul>`, `<li>`, `<dl>`, `<dt>`, `<dd>` 등)를 그대로 출력

파서 변경:
- 각 블록(FetchBlock, ActionBlock, EachBlock)에 `Tag string` 필드 추가
- data-* 속성이 붙은 요소의 태그명을 보존

```
현재: <div> ... </div>
개선: <section> ... </section>
```

### 2. Tailwind 클래스 보존

**현재**: class 속성 전부 누락
**개선**: HTML의 `class`를 JSX의 `className`으로 변환하여 보존

파서 변경:
- 모든 구조체에 `ClassName string` 필드 추가
- `class` 속성값을 그대로 수집

```tsx
// 현재
<div>

// 개선
<section className="flex-1 p-4">
```

### 3. 텍스트 콘텐츠 보존

**현재**: 버튼 텍스트가 전부 "제출", empty 상태 메시지가 "데이터가 없습니다"로 고정
**개선**: HTML 원본의 텍스트 노드를 보존

파서 변경:
- ActionBlock에 `SubmitText string` 추가 (button[type=submit]의 텍스트)
- StateBind에 `Text string` 추가 (data-state 요소의 텍스트 콘텐츠)

```tsx
// 현재
<button type="submit">제출</button>
<p>데이터가 없습니다</p>

// 개선
<button type="submit">예약하기</button>
<p>예약이 없습니다</p>
```

### 4. input 속성 보존

**현재**: placeholder, type만 부분 보존
**개선**: placeholder, type, 기타 HTML 속성을 모두 보존

파서 변경:
- FieldBind에 `Placeholder string` 추가
- FieldBind에 `Attrs map[string]string` 추가 (범용)

```tsx
// 현재
<input type="email" {...loginForm.register('Email')} />

// 개선
<input type="email" placeholder="이메일" className="w-full px-3 py-2 border rounded" {...loginForm.register('Email')} />
```

### 5. 필드 없는 action → 버튼 직접 호출

**현재**: DeleteRoom처럼 필드 없는 action도 form + useForm 생성
**개선**: 필드가 없으면 form 없이 button onClick으로 처리

```tsx
// 현재
const deleteRoomForm = useForm()
const deleteRoomMutation = useMutation({ ... })
<form onSubmit={deleteRoomForm.handleSubmit(...)}>
  <button type="submit">제출</button>
</form>

// 개선
const deleteRoomMutation = useMutation({ ... })
<button onClick={() => deleteRoomMutation.mutate({})} className="...">
  스터디룸 삭제
</button>
```

판단 기준: `len(action.Fields) == 0`

### 6. data-state 조건부 렌더링 구현

**현재**: `canCancel`, `canDelete` 등 plain 상태가 주석으로만 남음
**개선**: 상태별 렌더링 로직 생성

규칙:
- `<field>.empty` → `data?.field?.length === 0` (구현됨)
- `<field>.loading` → `fieldLoading` (추가)
- `<field>.error` → `fieldError` (추가)
- plain 상태 → 부모 fetch 응답에서 boolean 필드로 추론하거나, 별도 상태 변수 선언

```tsx
// 현재
{/* data-state: canCancel */}

// 개선 (fetch 응답에 canCancel이 있을 때)
{getReservationData.canCancel && (
  <footer className="mt-8 pt-4 border-t">
    <button onClick={() => cancelReservationMutation.mutate({})}>예약 취소</button>
  </footer>
)}
```

data-state 내부에 data-action이 있는 경우, 해당 action은 조건부 블록 안에 렌더링한다.

### 7. mutation onSuccess 범위 한정

**현재**: `queryClient.invalidateQueries()` — 전체 캐시 무효화
**개선**: 관련 쿼리만 무효화

규칙: 같은 페이지의 fetch operationId를 queryKey로 무효화

```tsx
// 현재
onSuccess: () => {
  queryClient.invalidateQueries()
}

// 개선
onSuccess: () => {
  queryClient.invalidateQueries({ queryKey: ['ListMyReservations'] })
}
```

구현: GeneratePage에서 페이지 내 fetch operationId 목록을 mutation 템플릿에 전달

### 8. 타입 생성

**현재**: `(data: any)` 전부
**개선**: OpenAPI schema에서 타입 추출

두 가지 접근:
- **A. 인라인 타입**: 코드젠 시 request/response 타입을 TSX 파일 상단에 interface로 생성
- **B. 공유 타입 파일**: `artifacts/frontend/types.ts`에 모든 스키마 타입을 생성하고 import

B안이 SSOT 원칙에 부합 (OpenAPI → 타입 파일 → 컴포넌트에서 import)

```tsx
// types.ts (자동 생성)
export interface CreateReservationRequest {
  RoomID: number
  StartAt: string
  EndAt: string
}

// my-reservations-page.tsx
import { CreateReservationRequest } from './types'
const createReservationMutation = useMutation({
  mutationFn: (data: CreateReservationRequest) => api.CreateReservation(data),
})
```

## 구현 우선순위

| 순위 | 항목 | 이유 | 난이도 |
|---|---|---|---|
| 1 | HTML 구조 + Tailwind 보존 | 시각적 차이가 가장 큼. 파서 수정 필요 | 중 |
| 2 | 텍스트 콘텐츠 보존 | 버튼/메시지 텍스트. 파서 수정 필요 | 소 |
| 3 | input 속성 보존 | placeholder, className. 파서 수정 필요 | 소 |
| 4 | 필드 없는 action → 버튼 | 템플릿만 수정 | 소 |
| 5 | data-state 구현 | 조건부 렌더링. 템플릿 수정 | 중 |
| 6 | onSuccess 범위 한정 | 템플릿만 수정 | 소 |
| 7 | 타입 생성 | 새 생성기 + OpenAPI 파싱 확장 | 대 |

1~3은 파서(Phase1)의 정보 수집량을 늘리는 공통 작업이므로 한 번에 진행한다. 4~6은 템플릿(Phase2)만 수정하면 된다. 7은 별도 Phase로 분리할 수 있다.

## 기대 결과

개선 후 코드젠 출력 예시 (my-reservations-page.tsx):

```tsx
'use client'

import React from 'react'
import { useQuery, useMutation, useQueryClient } from '@tanstack/react-query'
import { useForm } from 'react-hook-form'
import { api } from '@/lib/api'
import { CreateReservationRequest } from './types'
import DatePicker from '@/components/DatePicker'

export default function MyReservationsPage() {
  const queryClient = useQueryClient()

  const { data: listMyReservationsData, isLoading: listMyReservationsDataLoading, error: listMyReservationsDataError } = useQuery({
    queryKey: ['ListMyReservations'],
    queryFn: () => api.ListMyReservations(),
  })

  const createReservationForm = useForm<CreateReservationRequest>()
  const createReservationMutation = useMutation({
    mutationFn: (data: CreateReservationRequest) => api.CreateReservation(data),
    onSuccess: () => {
      queryClient.invalidateQueries({ queryKey: ['ListMyReservations'] })
    },
  })

  return (
    <main className="max-w-4xl mx-auto p-6">
      <header className="flex justify-between items-center mb-6">
        <h1 className="text-2xl font-bold">내 예약</h1>
      </header>

      {listMyReservationsDataLoading && <div>로딩 중...</div>}
      {listMyReservationsDataError && <div>오류가 발생했습니다</div>}
      {listMyReservationsData && (
        <section className="mb-8">
          <ul className="space-y-3">
            {listMyReservationsData.reservations?.map((item, index) => (
              <li key={index} className="flex justify-between items-center p-4 border rounded">
                <div>
                  <span className="font-semibold">{item.RoomID}</span>
                  <span className="ml-2 text-sm text-gray-500">{item.StartAt}</span>
                  <span className="text-sm text-gray-400">~</span>
                  <span className="text-sm text-gray-500">{item.EndAt}</span>
                </div>
                <span className="px-2 py-1 text-sm rounded bg-gray-100">{item.Status}</span>
              </li>
            ))}
          </ul>
          {listMyReservationsData.reservations?.length === 0 && (
            <p className="text-gray-400 text-center py-8">예약이 없습니다</p>
          )}
        </section>
      )}

      <section className="p-6 bg-gray-50 rounded-lg">
        <h2 className="text-lg font-semibold mb-4">새 예약</h2>
        <form onSubmit={createReservationForm.handleSubmit((data) => createReservationMutation.mutate(data))} className="space-y-4">
          <input type="number" placeholder="스터디룸 번호" className="w-full px-3 py-2 border rounded" {...createReservationForm.register('RoomID', { valueAsNumber: true })} />
          <div className="grid grid-cols-2 gap-4">
            <DatePicker {...createReservationForm.register('StartAt')} />
            <DatePicker {...createReservationForm.register('EndAt')} />
          </div>
          <button type="submit" className="w-full py-2 bg-blue-500 text-white rounded">예약하기</button>
        </form>
      </section>
    </main>
  )
}
```
