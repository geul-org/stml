# 수정지시서 004 실행결과

## 처리 방식

지시서가 요청한 `x-include` 런타임 이름 변환 대신, `data-include` 속성 자체를 STML에서 완전 제거하여 근본적으로 해결했다.

## 근거

- `x-include`는 model/SQL 계층(fullend)을 위한 선언이며, STML 프론트엔드 선언과 무관
- `data-include`는 UI에 아무 흔적도 남기지 않고 API 호출 파라미터만 바꾸는 인프라 관심사
- `data-paginate`/`data-sort`/`data-filter`는 UI 요소(버튼, 토글, 입력)를 생성하지만, `data-include`는 그렇지 않음
- 제거하면 런타임 이름 변환 문제 자체가 소멸
